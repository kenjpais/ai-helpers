# Agentic-Docs Plugin Structure

Complete directory layout for the agentic-docs plugin.

## Directory Tree

```
plugins/agentic-docs/
├── README.md                           # Plugin documentation
├── STRUCTURE.md                        # This file
│
├── config/
│   └── validation.yaml                 # User-configurable validation parameters
│
├── skills/
│   ├── create/
│   │   └── SKILL.md                    # /agentic-docs:create skill definition
│   ├── create-knowledge-graph/
│   │   └── SKILL.md                    # /agentic-docs:create-knowledge-graph skill
│   ├── validate/
│   │   └── SKILL.md                    # /agentic-docs:validate skill definition
│   └── evaluate/
│       └── SKILL.md                    # /agentic-docs:evaluate skill definition
│
└── lib/                                # Python utilities (no LLM logic)
    ├── __init__.py
    │
    ├── generators/                     # File-specific generators
    │   ├── __init__.py
    │   ├── design_generator.py         # DESIGN.md generation
    │   ├── development_generator.py    # DEVELOPMENT.md generation
    │   ├── testing_generator.py        # TESTING.md generation
    │   ├── reliability_generator.py    # RELIABILITY.md generation
    │   ├── security_generator.py       # SECURITY.md generation
    │   ├── quality_score_generator.py  # QUALITY_SCORE.md generation
    │   ├── core_beliefs_generator.py   # core-beliefs.md generation
    │   ├── component_architecture_generator.py
    │   ├── data_flow_generator.py
    │   ├── glossary_generator.py
    │   └── ...                         # Other file generators
    │
    ├── validators/                     # Validation logic
    │   ├── __init__.py
    │   ├── config_loader.py            # Load validation.yaml (immutable)
    │   ├── structure_validator.py      # Directory structure validation
    │   ├── navigation_validator.py     # Navigation depth validation
    │   ├── line_budget_validator.py    # Line budget validation
    │   ├── link_validator.py           # Broken link detection
    │   ├── quality_score_calculator.py # Quality score calculation
    │   └── validation_reporter.py      # Generate validation reports
    │
    ├── metrics/                        # Session metrics and analytics
    │   ├── __init__.py
    │   ├── session_scraper.py          # Scrape Claude Code session logs
    │   ├── execution_metrics.py        # Track execution time, tools used
    │   └── workflow_analytics.py       # AI-docs workflow analytics
    │
    └── logging/                        # Enhanced logging
        ├── __init__.py
        ├── logger.py                   # Structured logging
        ├── cli_formatter.py            # CLI-friendly output
        └── file_logger.py              # Timestamped log files
```

## Command to Skill Mapping

| Command | Skill File | Description |
|---------|-----------|-------------|
| `/agentic-docs:create` | `skills/create/SKILL.md` | Generate complete documentation |
| `/agentic-docs:create-knowledge-graph` | `skills/create-knowledge-graph/SKILL.md` | Generate knowledge graph |
| `/agentic-docs:validate` | `skills/validate/SKILL.md` | Validate documentation quality |
| `/agentic-docs:evaluate` | `skills/evaluate/SKILL.md` | Test with coding scenarios |

## Key Principles

### 1. Separation of Concerns
- **Skills (.md files):** High-level orchestration, LLM reasoning
- **Lib (Python code):** Deterministic operations, no LLM logic

### 2. Configuration-Driven
- All validation parameters in `config/validation.yaml`
- Agent cannot modify configuration during execution
- Users modify behavior by editing YAML

### 3. Observable
- Every command logs data sources, tools, skills
- Timestamped log files for every execution
- Execution metrics displayed after each command

### 4. File-Specific Generation
- Each documentation file has dedicated generator
- Promotes modularity and testability
- Easy to modify individual file generation logic

## Data Flow

### Create Command
```
User invokes: /agentic-docs:create
    ↓
skills/create/SKILL.md orchestrates
    ↓
For each file in framework:
    ↓
Call lib/generators/<file>_generator.py
    ↓
Generator analyzes repository
    ↓
Generator creates content
    ↓
File written to agentic/
    ↓
Logged to CLI + file
    ↓
Metrics displayed
```

### Validate Command
```
User invokes: /agentic-docs:validate
    ↓
skills/validate/SKILL.md orchestrates
    ↓
lib/validators/config_loader.py loads validation.yaml (immutable)
    ↓
For each validation check:
    ↓
Call appropriate validator (structure, navigation, line budget, etc.)
    ↓
Validators report pass/fail
    ↓
lib/validators/validation_reporter.py generates summary
    ↓
Report displayed + logged
    ↓
Metrics displayed
```
