# Agentic-Docs: Create

**Purpose**: Orchestrate complete agentic documentation generation by coordinating specialized file-generation skills.

**Key Principle**: This is a thin orchestrator with NO generation logic. Each file is generated by invoking its dedicated skill.

## Quick Reference

```
Workflow: Initialize → Structure → Generate (4 phases) → Validate → Evaluate → Report
Duration: ~5-10 minutes for typical repository
Output: 18 documentation files across agentic/ directory + evaluation suite
```

## Input

```bash
/agentic-docs:create [<repo-path>] [--skip-evals] [--parallel]
```

**Arguments**:
- `<repo-path>`: Target repository (defaults to current directory)
- `--skip-evals`: Skip evaluation suite generation (faster)
- `--parallel`: Run independent skills in parallel (experimental, faster but uses more resources)

### Phase 1: Create Directory Structure

**REQUIRED**: Execute the Python structure generator script (NOT mkdir).

**Command** (execute via Bash tool):
```bash
# Robust structure generator execution
STRUCTURE_GEN="${CLAUDE_PLUGIN_ROOT}/lib/generators/structure_generator.py"
[ ! -f "$STRUCTURE_GEN" ] && STRUCTURE_GEN=$(find ~/.claude/plugins -type f -path "*/agentic-docs/lib/generators/structure_generator.py" 2>/dev/null | head -1)

if [ -z "$STRUCTURE_GEN" ] || [ ! -f "$STRUCTURE_GEN" ]; then
  echo "ERROR: structure_generator.py not found" >&2
  exit 2
fi

cd <repo-path> && python3 "$STRUCTURE_GEN" .
```

Replace `<repo-path>` with actual repository path.

**Why script over mkdir**: Creates 13 directories + 18 files atomically, validates structure, prevents partial creation errors.

**Expected Output**:
```
📁 Creating agentic/ directory structure in: .
  ✅ Created: agentic/
  ... (13 directories, 18 files)
✅ Structure validation PASSED
```

**Error Handling**:
- Exit code ≠ 0: CRITICAL failure, abort entire workflow
- Partial creation: Script handles cleanup automatically
- Already exists: Script validates existing structure

### Phase 6: Validate Output

**Skill**: `/agentic-docs:validate <repo-path>`

**Checks**:
- All required files present (18 files)
- Line budgets enforced (AGENTS.md ≤150 lines, others per guidelines)
- No broken documentation links
- Navigation depth ≤3 hops
- Quality score ≥70/100

**Error Handling**:
- Validation warnings: Continue, report in summary
- Validation errors (missing critical files): Mark as FAILED, report details
- Cannot run validator: Mark as SKIPPED, warn user

### Phase 7: Generate Evaluation Suite

**Auto-invoked** unless `--skip-evals` flag provided.

**Skill**: `/agentic-docs:generate-evals <repo-path>`

**Purpose**: Generate repository-specific `promptfooconfig.yaml` with tests tailored to the repository's patterns.

**Output**:

- `EVALUATION.md` - Evaluation documentation
- Test scenarios: 2-3 navigation + 1-2 authoring + 3-5 anti-pattern

**Error Handling**: Non-critical - warn if fails, continue to Phase 8.

## Error Handling Strategy

### Critical vs Non-Critical Failures

**CRITICAL** (abort workflow):
- `structure_generator.py` fails (exit code ≠ 0)
- `generate-agents-md` fails (AGENTS.md is primary entry point)
- `generate-architecture-md` fails (ARCHITECTURE.md required for navigation)
- Repository path invalid/inaccessible

**NON-CRITICAL** (log, continue, report at end):
- Individual core/design/domain doc generation fails
- Validation warnings (not errors)
- Evaluation suite generation fails
- Quality score below threshold

### Error Recovery Procedure

**When critical skill fails**:
1. Log failure with full error details
2. Display clear error message to user:
   ```
   ❌ CRITICAL FAILURE: <skill-name> failed

   Error: <error-message>

   Workflow aborted. Partial documentation may exist in agentic/ directory.
   Log file: <log-path>

   Troubleshooting:
   - Review log file for details
   - Check repository structure matches expected patterns
   - Try running failed skill individually: /<skill-name> <repo-path>
   ```
3. Exit with non-zero status
4. Do NOT attempt to continue workflow

**When non-critical skill fails**:
1. Log failure with error details
2. Add to `failed_skills` list in metrics
3. Continue with remaining skills
4. Report failures in final summary:
   ```
   ⚠️  Warnings:
   - generate-reliability-md failed: <error>
   - generate-security-md failed: <error>

   Completed: 10/12 skills (83%)
   ```

**Retry Logic**: Do NOT automatically retry failed skills. User can manually retry individual skills if needed.

### Partial Cleanup

If critical failure occurs after structure creation:
- Leave partial documentation in place (DO NOT delete)
- User can inspect partial output
- User can re-run create skill (structure generator handles existing files)

## Performance Optimization

### Parallel Execution (Experimental)

When `--parallel` flag provided:

**Phase 2-4 (parallelizable skills)**:
```bash
# Launch all Phase 2-4 skills as background subagents
# Wait for all to complete before Phase 5
```

**Benefits**:
- ~40-60% faster (6-12 independent skills in parallel)
- Better resource utilization

**Trade-offs**:
- Higher memory usage (multiple Claude sessions)
- More complex error handling
- Harder to debug failures

**Recommendation**: Use parallel mode for large repositories (>100 files). Sequential for smaller repos or debugging.

### Execution Time Estimates

| Phase | Sequential | Parallel | Notes |
|-------|-----------|----------|-------|
| 0-1: Init + Structure | ~30s | ~30s | Not parallelizable |
| 2-4: Generate Docs | 4-8 min | 2-3 min | 12 skills |
| 5: Entry Points | 1-2 min | 1-2 min | Sequential dependency |
| 6-8: Validate + Summary | ~1 min | ~1 min | Quick |
| **Total** | **6-12 min** | **4-7 min** | Varies by repo size |
