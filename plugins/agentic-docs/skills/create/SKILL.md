# Agentic-Docs: Create (Orchestrator)

**Trigger**: `/agentic-docs:create`  
**Purpose**: Orchestrate complete agentic documentation generation by coordinating file-specific generation skills

## Architecture

This skill is a **thin orchestrator** that:
- Initializes logging and directory structure
- Coordinates file-specific generation skills (12 dedicated skills)
- Validates output quality
- Reports final metrics

**Key Principle**: This skill contains NO generation logic. Each file is generated by invoking its dedicated skill.

## Input

**Repository Path** (optional - defaults to current directory)

```
/agentic-docs:create [<repo-path>]
```

## Error Handling

If any skill fails:
1. Log the failure
2. Continue with remaining skills (best-effort)
3. Report which skills failed at the end
4. Exit with non-zero status if any critical skill failed

**Critical Skills** (must succeed):
- structure_generator.py
- generate-agents-md
- generate-architecture-md

**Non-Critical Skills** (warn on failure):
- All other generation skills

## Logging Requirements

Every run must log to timestamped file using **`lib/metrics_logger.py`**:
- Command invoked
- Repository path
- Timestamp
- Each skill invoked (with timing)
- Data sources used (aggregated)
- Tools used (aggregated)
- Files created (list)
- Validation result
- Final metrics
- Total duration

**Implementation**: The skill uses `MetricsLogger` from `lib/metrics_logger.py` to provide:
- Real-time logging to CLI (all tool/skill invocations logged as they happen)
- Persistent logging to timestamped log file (`logs/agentic-docs-create-{timestamp}.log`)
- Post-execution metrics summary (duration, tool usage breakdown, skill usage, data sources)
- Metrics JSON export (`logs/agentic-docs-create-{timestamp}.metrics.json`)

See [lib/METRICS_LOGGER_USAGE.md](../../lib/METRICS_LOGGER_USAGE.md) for complete usage guide.
