# Agentic-Docs Plugin

Complete lifecycle management for AI-friendly documentation in OpenShift/Kubernetes repositories.

## Overview

This plugin provides comprehensive agentic documentation capabilities for OpenShift repositories, following the Agent Knowledge Framework defined in [agentic-docs-guide](https://github.com/Prashanth684/agentic-docs-guide/blob/main/AGENTIC_DOCS_FRAMEWORK.md).

## Commands

All commands use the `/agentic-docs:` namespace:

### `/agentic-docs:create`

Generate complete agentic documentation for a repository.

**Usage:**
```bash
/agentic-docs:create [<repo-path>]
```

**What it does:**
- Analyzes repository structure
- Generates documentation following the framework structure
- Creates all required files (DESIGN.md, DEVELOPMENT.md, TESTING.md, etc.)
- Each file is generated by a dedicated skill/prompt
- Logs all data sources, tools, and skills used

**Input:** Repository files only (no GitHub/JIRA ingestion)

**Output:**
- Complete agentic documentation in `agentic/` directory
- CLI logging with real-time progress
- Timestamped log file in `logs/`

### `/agentic-docs:create-knowledge-graph`

Generate knowledge graph from existing agentic documentation.

**Usage:**
```bash
/agentic-docs:create-knowledge-graph [<repo-path>]
```

**What it does:**
- Reads existing agentic documentation
- Generates NetworkX knowledge graph
- Creates graph.json in `agentic/knowledge-graph/`

**Requirements:** Must be run AFTER `/agentic-docs:create`

### `/agentic-docs:validate`

Validate existing agentic documentation against quality standards.

**Usage:**
```bash
/agentic-docs:validate [<repo-path>]
```

**What it does:**
- Validates directory structure
- Checks navigation depth (≤3 hops)
- Verifies line budgets
- Checks for broken links
- Calculates quality score (0-100)

**Configuration:** All validation parameters are defined in `config/validation.yaml`

**Output:**
- Validation summary report
- Repository name and timestamp
- Pass/fail status for each check

### `/agentic-docs:evaluate`

Test documentation quality with coding scenarios.

**Usage:**
```bash
/agentic-docs:evaluate [<repo-path>]
```

**What it does:**
- Spawns coding sub-agents with benchmark tasks
- Tests documentation usability
- Evaluates execution plans
- Displays agent execution flow

**Output:**
- Primary agent and sub-agents display
- Transparent execution flow logs
- Data flow between components
- Timestamped evaluation logs

## Architecture

### File-Specific Generation

Each file in the agentic documentation structure is generated by its own dedicated skill:

- `DESIGN.md` → `lib/generators/design_generator.py`
- `DEVELOPMENT.md` → `lib/generators/development_generator.py`
- `TESTING.md` → `lib/generators/testing_generator.py`
- `RELIABILITY.md` → `lib/generators/reliability_generator.py`
- `SECURITY.md` → `lib/generators/security_generator.py`
- `QUALITY_SCORE.md` → `lib/generators/quality_score_generator.py`
- `core-beliefs.md` → `lib/generators/core_beliefs_generator.py`
- `component-architecture.md` → `lib/generators/component_architecture_generator.py`
- And so on...

### Validation Configuration

All validation parameters are user-configurable via `config/validation.yaml`. The agent **cannot** modify these parameters during execution (strict immutability enforced).

### Logging

Every command execution produces:
- **CLI output:** Real-time progress and results
- **Timestamped log file:** Complete execution trace
- **Execution metrics:** Performance and usage analytics

### Metrics

After every command execution, the system displays:
- Execution time
- Tools and skills used
- Files accessed
- AI-docs workflow analytics

### Session Metrics Scraping

The plugin includes a session scraper that extracts file access patterns from Claude Code session logs:

```python
from lib.metrics.session_scraper import SessionScraper

# Scrape all sessions
scraper = SessionScraper()
telemetry_list = scraper.scrape_all_sessions()

# Export to JSON
scraper.export_to_json(telemetry_list, Path("session_telemetry.json"))

# Aggregate metrics
metrics = scraper.aggregate_metrics(telemetry_list)
print(f"Total sessions: {metrics['total_sessions']}")
print(f"Avg accesses/session: {metrics['avg_accesses_per_session']:.1f}")
```

**Metrics extracted:**
- File access patterns (which docs accessed, in what order)
- Entry points (AGENTS.md vs direct search)
- Navigation sequences with timestamps
- Retrieval timing (time spent retrieving vs generating)
- Most accessed documentation files

**Data source:** `~/.claude/projects/**/*.jsonl` (Claude Code session logs)

## Quality Guarantees

### Navigation Constraint
- All information ≤3 hops from AGENTS.md
- No orphaned documents
- Clear navigation paths

### Context Budget
- AGENTS.md: ≤150 lines
- Component docs: ≤100 lines each
- Concept docs: ≤75 lines each

### Quality Score (0-100)
- Coverage: 40 points (≥80% components documented)
- Freshness: 20 points (updated <90 days)
- Completeness: 20 points (all required files)
- Linkage: 10 points (no broken links)
- Navigation: 10 points (≤3 hop depth)
- **Pass threshold: ≥70/100**

## Configuration

### Validation Parameters

Edit `config/validation.yaml` to customize:
- Navigation depth (default: 3 hops)
- Line budgets (AGENTS.md: 150, components: 100, concepts: 75)
- Required directory structure
- Quality score thresholds
- Validation checks to enable/disable

### Example: Change Navigation Depth

```yaml
# config/validation.yaml
navigation:
  max_hops: 4  # Changed from default 3
```

## Development

### Adding a New File Generator

1. Create generator in `lib/generators/<filename>_generator.py`
2. Implement generation logic
3. Add skill invocation in `skills/create/SKILL.md`
4. Test with `/agentic-docs:create`

### Testing

```bash
# Test create command
/agentic-docs:create /path/to/test-repo

# Test validation
/agentic-docs:validate /path/to/test-repo

# Test evaluation
/agentic-docs:evaluate /path/to/test-repo

# Test knowledge graph
/agentic-docs:create-knowledge-graph /path/to/test-repo
```
